# ─── How to report ─────────────────────────────────────────────
# Choose any one of three channels — all reach our security team:
#
#   1. Email security@b1n0.com directly.
#   2. Submit a private advisory via GitHub:
#      https://github.com/kimallenchee/b1n0/security/advisories/new
#   3. Submit a public-but-coordinated report via OpenBugBounty:
#      https://www.openbugbounty.org/
#
# Include: short description, steps to reproduce (or minimal PoC),
# the impact you believe a malicious actor could achieve, and your
# name/handle for acknowledgment (optional — pseudonymous OK).
#
# We commit to:
#   - Acknowledging your report within 5 business days.
#   - Triaging and providing a preliminary severity within 10 business days.
#   - Keeping you informed of remediation progress.
#   - Crediting you on https://www.b1n0.com/confianza#agradecimientos
#     unless you ask to remain anonymous.
#
# Please do not publicly disclose the issue until we've had a
# reasonable opportunity to remediate (default coordinated-disclosure
# window: 90 days post-fix).

# ─── In scope ──────────────────────────────────────────────────
# - https://www.b1n0.com  (production app)
# - https://b1n0.com      (apex redirect)
# - Supabase project bebdvsdiqlruqzmkvmgy (auth, RLS, RPCs, edge functions)
# - The b1n0 PWA / mobile experience
# - Third-party integrations as used by b1n0 (Didit, Resend, Pagadito, etc.)
#   only insofar as the integration enables a vulnerability in b1n0 itself

# ─── Out of scope ──────────────────────────────────────────────
# - Social engineering of Tres33 employees, contractors, or partners
# - Physical attacks against Tres33 infrastructure or staff
# - Denial-of-service / volumetric attacks
# - Rate-limit bypass without a downstream impact
# - Findings from automated scanners without demonstrable impact
# - Vulnerabilities in third-party services themselves (report to the vendor)
# - Self-XSS or attacks requiring already-compromised client browsers
# - Missing best-practice headers on purely static / marketing surfaces
# - UI/UX bugs, broken links, content typos — soporte@b1n0.com instead
# - Email-spoofing reports against a domain with no inbound mail flow

# ─── Safe-harbor commitment ────────────────────────────────────
# Tres33 SAS de CV will not pursue legal action against researchers
# who act in good faith, stay within the scope above, avoid privacy
# violations and service disruption, and give us a reasonable window
# to fix issues before public disclosure. This program follows the
# disclose.io safe-harbor framework — see https://disclose.io/legal/
